# Water Sort Puzzle — 設計書

子供向けの水並び替えパズルゲーム。iPhone / FireHD のブラウザで動作する単一HTMLファイル。

---

## 1. ゲーム概要

### コンセプト
- カラフルな液体が混ざったボトルを並び替え、各ボトルを1色だけにするパズル。
- タップ操作のみ。直感的で子供でも遊べる。
- 30ステージを最初から自由に選択可能。

### プラットフォーム要件
| 端末 | 解像度目安 | 向き |
|------|-----------|------|
| iPhone (SE〜Pro Max) | 375×667〜430×932 | 縦 |
| iPad / Fire HD 8 | 1280×800 | 横/縦両対応 |
| Fire HD 10 | 1920×1200 | 横/縦両対応 |

`viewport` メタタグでスケール固定、`safe-area-inset` 対応、`-webkit-tap-highlight-color: transparent` で iOS のタップ青枠を消す。

## 3. ゲームルール

### 3.1 基本ルール
- 各ボトルは容量4セル（液体の塊が4つ重ねて入る）。難易度Hardは容量5。
- ボトルAをタップ → 選択（少し浮き上がる＋光る）
- 選択中にボトルBをタップ：
  - **注げる条件**：(1) Bが空、または (2) Bの一番上の色がAの一番上の色と一致し、かつBに空きがある
  - 注げる場合：Aの上から連続する同色の塊を、Bが満杯になるまで全て移す
  - 注げない場合：振動アニメ＋エラー音、選択解除
- もう一度同じボトルをタップ：選択解除

### 3.2 クリア条件
- 全ボトルが「空」または「同色4つ（容量分）」のいずれか。

### 3.3 補助機能
- **🔄 リセット**：そのステージの初期状態に戻す（確認ダイアログなし、ワンタップ）
- **↩️ 戻る**：直前の1手を取り消す。最大10手まで履歴保持
- **➕ ボトル追加**：空ボトルを1つ追加（1ステージにつき1回のみ）

### 4.2 ステージ生成アルゴリズム
事前生成（ハードコード）方式。再現性のため固定シードで生成する。

**逆順生成法**：
1. 「クリア状態」（各ボトルが単色4つ）を初期状態とする。
2. ランダムに「逆注ぎ」を N 回繰り返す（Nは難易度に比例）。
   - ランダムに非空ボトルAを選び、別のボトルBを選ぶ
   - AからBへ、Bが受け取れる量だけ「移す」（ただし通常の注ぎルールは無視。色は変えてよい）
3. 結果を初期配置として採用。最低でも10手以上必要なものをフィルタする。

→ **必ず解ける配置**を保証できる。30個分を事前計算してJSON配列としてコード内に埋め込む。

## 6. アニメーション / クリック演出

### 6.1 タップ演出
- **リップル**：タップ位置から円が広がる（CSS `::after` ＋ `animation: ripple 0.5s`）
- **ボトル選択**：`transform: translateY(-12px) scale(1.05)` ＋ `box-shadow` で発光
- **無効タップ**：左右に揺れる（`shake 0.3s`）

### 6.2 注ぎアニメ
1. ソースボトルが傾く（`transform: rotate(-60deg)`、0.3s）
2. 注ぎ口から液体が円弧を描いて落ちる（SVG path animation または DOM の小さな水滴を絶対配置で連続生成）
3. ターゲットボトルの液体が下から上に伸びる（`height` トランジション、各セル0.15s）
4. ソースボトルが戻る
5. 着水時に小さな波紋（白い円が広がって消える）

### 6.3 液体の細部
- 各液体セルは `gradient` で上部にハイライト・下部に影
- 表面に「揺れ」アニメ（`clip-path` を `polygon` で波形に変えて2秒ループ）
- 移動時は気泡が3〜5個立ち上がる

### 6.4 クリア演出
- 紙吹雪：50個のSVG片を上から落とす（`requestAnimationFrame` ベース）
- 装飾ボトル中央から放射状に星マーク
- 「クリア！」テキストがバウンスで登場
- 1.5秒後にモーダル表示

### 6.5 ホーム画面のステージタイル
- ホバー/タップで `scale(1.1)`
- タップで縮んでから次画面へフェード（0.3s）

## 7. サウンド設計

すべて Web Audio API で動的生成（音源ファイル不要）。

### 7.1 効果音 (SE)
| イベント | 音 | 実装 |
|---------|----|------|
| ボトル選択 | 短いプッ音 | サイン波 880Hz、0.05s |
| 注ぎ開始 | 水のサラサラ音 | ホワイトノイズ＋バンドパス、0.3s |
| 着水 | ポチャ | 三角波 220Hz→110Hz スイープ、0.1s |
| セル完成 | キラン | 三角波 1320Hz→1760Hz、0.15s |
| 無効タップ | ブッ | のこぎり波 110Hz、0.1s |
| クリア | ファンファーレ | 3音アルペジオ C-E-G、各0.15s |
| ボタンタップ | クリック | 矩形波 660Hz、0.03s |

### 7.2 BGM
- シンプルな8音アルペジオを4小節、`OscillatorNode` 群でループ
- 音量は SE の40%
- ホーム画面：明るい長調
- ゲーム中：穏やかな短調寄り
- ON/OFFと音量はゲーム内で記憶（メモリ上のみ、リロードでリセット）

### 7.3 設定の制約（保存しない方針）
進捗保存しない方針のため、サウンド設定もページリロードで初期値に戻る。代わりに「セッション中は維持」で良い。

### 8.4 タッチ最適化
- `touchstart` で反応（`click` の300ms遅延を排除）
- 全インタラクティブ要素は最低 44×44px
- スクロール禁止：`body { overflow: hidden; touch-action: manipulation; }`
- iOS のダブルタップズーム抑止：`viewport` で `maximum-scale=1`

### 8.5 レスポンシブ
- CSS変数 `--vh` をJSで `window.innerHeight * 0.01` に設定（iOS Safari の100vh問題対策）
- ボトルサイズはCSS Grid + `clamp()` で可変
- 画面サイズに応じてボトル列数を調整：
  - 縦長(iPhone)：3〜4列
  - 横長(FireHD)：6〜8列
