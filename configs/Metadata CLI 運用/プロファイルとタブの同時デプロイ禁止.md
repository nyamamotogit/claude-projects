# プロファイルとタブの同時デプロイ禁止（2段階実施）

**取得元**: sf-create-objects スキル / 2026-05-11  
**Salesforce API**: 63  
**構成**: 状況 → 注意点 → 対処

---

## 状況

カスタムオブジェクト・タブ・項目・プロファイルのタブ可視性をまとめて 1 回のデプロイで送ろうとした場合。

---

## 注意点

### エラー内容

タブとプロファイルを同一デプロイリクエストに含めると、プロファイル側がタブ参照で失敗する：

> `In field: tab - no CustomTab named ObjectName__c found`

### 真因

1 回のデプロイ内では依存解決が順序保証されない。プロファイルの `<tabVisibilities>` 検証時点でタブがまだ org に存在しないと弾かれる。

### 誤診されやすさ

- タブ XML 自体に誤りがあると思い込み、`<motif>` や `<customObject>` を疑い始める
- プロファイル側のタブ可視性記述を疑う
- どちらも正しいので原因に辿り着かず時間を溶かす

---

## 対処

**オブジェクト/タブ/項目を先にデプロイ → 成功確認後にプロファイルをデプロイ**：

```bash
# Step 1: オブジェクト・タブ・項目
sf project deploy start \
  --source-dir force-app/main/default/objects \
  --source-dir force-app/main/default/tabs \
  --target-org target-org

# Step 2: 成功を確認してからプロファイル
sf project deploy start \
  --source-dir force-app/main/default/profiles \
  --target-org target-org
```

---

## ついでの注意: タブ XML には `<object>` 要素は書かない

カスタムタブ XML は**ファイル名（`ObjectName__c.tab-meta.xml`）でオブジェクトが特定される**ため、内部に `<object>` タグを記述すると検証エラーになる。最小構成は以下：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<CustomTab xmlns="http://soap.sforce.com/2006/04/metadata">
    <customObject>true</customObject>
    <motif>Custom62: Handsaw</motif>
</CustomTab>
```

`<motif>` 値は既存タブを retrieve して同系統のものをコピーするのが最確実。
